üîë 1. API Health Check

First, verify the API is running:

üìã 2. Get Campaign Types

See what campaign types are available:

ü§ñ 3. Generate Your First Campaign

Generate a campaign with a simple description:

curl -X POST "http://localhost:8000/api/v1/campaigns/generate" \ -H "Content-Type: application/json" \ -d '{ "merchant_id": "test_001", "description": "Create a welcome campaign for new subscribers", "campaign_type": "welcome" }'

‚úÖ 4. Validate Campaign

Check if your campaign JSON is valid:

curl -X POST "http://localhost:8000/api/v1/campaigns/validate" \ -H "Content-Type: application/json" \ -d '{ "campaign_json": { "initialStepID": "step_001", "steps": [ { "id": "step_001", "type": "message", "content": "Welcome to our store! Enjoy 15% off with code WELCOME15." } ] } }'

‚ùì Common Issues & Solutions

Issue: Connection Refused

Cause: API server not running

Solution: Start the server:

python -m src.main

Issue: Campaign Generation Timeout

Cause: API processing takes 15-20 seconds normally

Solution: Increase timeout in your HTTP client

Issue: Invalid API Key

Cause: Missing or incorrect API keys

Solution: Check your .env file contains valid keys

  • OPENROUTER_API_KEY
  • COHERE_API_KEY

Issue: Validation Errors

Cause: Invalid campaign JSON structure

Solution: Use the validation endpoint to check structure

{ "initialStepID": "step_001", "steps": [ { "id": "step_001", "type": "message", "content": "Your message here", "events": [] } ] }
